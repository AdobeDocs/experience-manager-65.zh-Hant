# 商務整合架構GitHub發行概觀

## 發行日期：2020年11月

| GitHub | 版本 | 詳細發行說明 |
|:-------|:-----:|---------------------:|
| CIF連接器 | 1.6.0 | [發行說明](https://github.com/adobe/commerce-cif-connector/releases) |
| CIF核心元件 | 1.6.0 | [發行說明](https://github.com/adobe/aem-core-cif-components/releases) |
| CIF韋尼亞參考站 | 2020.12.01 | [發行說明](https://github.com/adobe/aem-cif-guides-venia/releases) |

### 新功能 {#what-is-new-november}

* 範本繼承已新增至特定類別頁面。 此功能可改善商業使用者的效率，因為所有子類別都可繼承為特定頂層類別建立的範本。

* 文尼亞參考店面已更新為頁尾使用體驗片段。 商業使用者可使用製作工具來編輯頁AEM尾。

### {#what-is-improved-november}的改善功能

* 結帳元件已改善，讓購物者能夠進入目的地國家，以允許美國以外地區的帳單／送貨地址。

* 導航元件擴展到水合Adobe客戶端資料層。

* 多個錯誤修正。

### 新功能 {#what-is-new-october}

* 新增「類別」浮動切換元件，讓商業使用者將此元件拖放至內容頁AEM面，以豐富商務資料的內容頁面。

* CIF核心元件通過發送商務資料來擴展Adobe客戶資料層。 Adobe用戶端資料層是一種標準化方法，用來收集資料並將資料傳送至數位分析和報告伺服器。 有關詳細資訊，請參閱[Adobe客戶端資料層](https://github.com/adobe/adobe-client-data-layer/wiki)。

* 產品詳細資訊和產品清單頁面已擴充，可自動填入從Magento管理UI中設定的SEO中繼資料（例如標題、中繼說明、中繼關鍵字）

* 商務摘要元件錯誤已修正。

* 支援Magento2.4.0架構的查詢。

* 新增帳戶資訊功能，讓購物者可更新個人資訊。

* 懶惰地載入為「產品」清單和「搜尋結果」頁面實作的分頁樣式，可讓開發人員將這些元件設定為將「載入更多」按鈕顯示為分頁樣式。

* 開發人員可以設定產品轉盤、相關產品和功能類別清單元件的HTML標籤，以遵循SEO最佳實務。

* 我的帳戶錯誤已修正。

### 新功能 {#what-is-new-june}

這是Adobe Experience Manager支援的第一個CIF核心元件版本。

* 在「產品清單」頁面和「搜尋結果」頁面新增產品排序，讓購物者可以根據相關性、價格和產品名稱進行排序。

* 新增類別篩選作為Facet，讓購物者可依據類別進行篩選。

* 已添加服務用戶映射作為安全要求的一部分，以確保通過服務用戶而不是直接操作ACL訪問/conf。 CIF核心元件現在必須使用服務用戶訪問配置。

### {#what-is-improved-june}的改善功能

* 「產品清單」頁面和「搜尋結果」頁面會顯示項目總數。 購物者套用篩選時，會更新項目數。

* 結合類別查詢與產品搜尋查詢，以最佳化多面搜尋。

* 頁面預覽的類別／產品挑選器榮譽cq:catalogPath。

* 多個錯誤修正。
