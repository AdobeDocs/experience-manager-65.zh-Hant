# [!DNL Adobe Experience Manager] 6.5.10.0 {#experience-manager-65100}

[!DNL Adobe Experience Manager] 6.5.10.0包括自2019年4月6.5版發佈以來發佈的新功能、客戶要求的關鍵增強功能以及效能、穩定性和安全性改進。 Service Pack已安裝在 [!DNL Adobe Experience Manager] 6.5

中介紹的關鍵功能和增強功能 [!DNL Adobe Experience Manager] 6.5.10.0是：

* **增強 [!DNL Content Fragment] 模型和編輯器**:現在，您可以使用嵌套方式為結構化內容建立複雜的自定義模型 [!DNL Content Fragment] 模型。 內容結構被模組化成基本元素，其被建模為子片段。 較高級片段引用這些子片段。 更多資料類型增強功能（如高級驗證規則）進一步增強了內容建模的靈活性 [!DNL Content Fragments]。 的 [!DNL Experience Manager] [!DNL Content Fragment] 編輯器支援通用編輯器會話中的嵌套片段結構，其中包括結構樹視圖和通過片段層次結構進行的頁籤式breadcrumb導航等增強功能。

* **GraphQL API[!DNL Content Fragments]**:新的GraphQL API是以JSON格式傳送結構化內容的標準方法。 GraphQL查詢允許客戶端僅請求相關內容項來呈現體驗。 這樣的選擇消除了需要在客戶端上進行內容分析的內容過傳遞（HTTP REST API的可能性）。 GraphQL架構是從 [!DNL Content Fragment] 模型和API響應以JSON格式進行。

* **GraphQL API[!DNL Content Fragments]**:要支援GraphQL API，「內容片段模型屬性」欄位中不再允許連字元。 如果任何「內容片段模型」屬性名稱中都存在連字元，則GraphQL查詢可能會返回不需要的結果。
屬性名稱僅允許以下字元：A-Za-z0-9_ 數字不能位於第一位置。

* **層次管理和將來預覽**:用戶現在具有訪問其內容結構的介面 [!DNL Experience Manager] 啟動，包括在啟動中添加和刪除頁面的功能。 此功能增強了 [!DNL Experience Manager] 啟動以建立針對將來發佈的內容版本。 [時間曲線特徵](/help/sites-authoring/working-with-page-versions.md#timewarp) 允許用戶將啟動作為未來內容狀態進行預覽。

* **已連接資產**: [!DNL Experience Manager] 延伸 [!DNL Connected Assets] 功能 [!DNL Dynamic Media] 影像。 請參閱 [使用連接的資產](/help/assets/use-assets-across-connected-assets-instances.md)。

* **連結共用選項以下載資產或格式副本**:當將資產和收藏作為連結共用時，用戶可以選擇是否允許下載原始資產或其格式副本，或者使用共用連結同時下載兩者。 此外，通過連結下載與他們共用的資產的用戶可以選擇只下載原始資產、只下載格式副本或兩者。 請參閱 [通過連結共用資產](/help/assets/link-sharing.md)。

   ![選項，僅允許下載原始資產，僅允許格式副本，或同時允許](/help/release-notes/assets/share-assets-as-link.png)

* **限制生成的子資產**:管理員可以限制 [!DNL Experience Manager] 為複合資產(如PDF、PowerPoint、InDesign和Keynote檔案)生成。 請參閱 [管理複合資產](/help/assets/managing-linked-subassets.md#generate-subassets)。

   ![限制子資產生成](/help/assets/assets/sub-asset-limit.png)


* **Camera Raw支援**:新 [!DNL Camera Raw] 包可用，支援 [!DNL Adobe Camera Raw] v10.4。請參閱 [處理影像 [!DNL Camera Raw]](/help/assets/camera-raw.md)。

* 內置儲存庫(Apache Jackrabbit Oak)已更新為1.22.8。

* **協助工具增強功能**:

   * [!DNL Dynamic Media] 為查看器提供了許多輔助功能增強。 請參閱 [[!DNL Dynamic Media] 更新](#dynamic-media-65100)。

   * 平台提供了一些輔助功能增強。 請參閱 [平台更新](#platform-65100)。

* **用戶體驗增強**:

   * [!DNL Experience Manager] 直接顯示資料夾下所有內容模型的清單，而內容作者不必在檔案結構中導航。 現在，該功能需要的點擊量更少，並且提高了創作效率。

   * 中的路徑欄位 [!DNL Sites] 編輯器允許作者從 [!DNL Content Finder]。

* 為 `GuideBridge#getGuidePath` API在 [!DNL AEM Forms]。

* 您現在可以使用Automated forms conversion服務 [用法語、德語、西班牙語、義大利語和葡萄牙語轉換PDF forms語](https://experienceleague.adobe.com/docs/aem-forms-automated-conversion-service/using/extending-the-default-meta-model.html#language-specific-meta-model) 到自適應窗體。

* **屬性瀏覽器中的錯誤訊息**：新增最適化表單屬性瀏覽器中每種屬性的錯誤訊息。這些訊息有助於了解欄位允許的值。

* **支援使用文字選項為JSON類型變數設定值**:可以使用文字選項在工作流的設定變數步驟中為JSON類型變數設AEM置值。 此常值選項可讓您使用字串形式來指定 JSON。

* [平台更新](/help/forms/using/aem-forms-jee-supported-platforms.md): [!DNL Adobe Experience Manager Forms] on JEE增加了對以下平台的支援：
   * [!DNL Adobe Acrobat 2020]
   * [!DNL Ubuntu 20.04]
   * [!DNL Open Office 4.1.10]
   * [!DNL Microsoft Office 2019]
   * [!DNL Microsoft Windows Server 2019]
   * [!DNL RHEL8]

以下是中提供的修復清單 [!DNL Experience Manager] 6.5.10.0版。

## [!DNL Sites] {#sites-65100}

* 在中鍵入時，焦點將移至另一個欄位 **[!UICONTROL 預設值]** 欄位 **[!UICONTROL 屬性]** 頁籤。

* 篩選時 [!DNL Content Fragment] 模型， [!DNL Experience Manager] 搜索返回所有節點 `cq:Template` 而不是僅返迴路徑和節點 [!DNL Content Fragment] 型號(SITES-1453)。
* [!DNL Content Fragments] 返回 `null` 資料夾狀態(SITES-1157)。
* [!DNL Experience Manager] 不允許用戶禁用和啟用 [!DNL Content Fragment] 型號(SITES-1088)。
* 當用戶移動、更名或刪除時 [!DNL Content Fragments] 或介質資產，引用 [!DNL Content Fragments] 未自動更新(SITES-196)。
* 將元件從一個頁面貼上到另一個頁面會產生JavaScript錯誤(NPR-37030)。
* 快速查看頁面屬性時，會開啟其他頁面的「頁面屬性」(NPR-37025)。
* 內容片段允許內容片段引用自身。 機械臂不支援該操作(NPR-36993)。
* 升級到Service Pack 9後，某些用戶無法在Experience Manager中移動資料夾，並在日誌中看到錯誤(SITES-1481)。
* 在編輯模式下調整佈局容器中元件的寬度時，觀察閃爍(NPR-36961)。
* 在推廣發射時，推廣發射的變化將與其他發射雙倍展開。 如果用戶推廣雙重展開發射，則雙重內容在源頁(NPR-36893)上反映。
* [!DNL Experience Manager] 如果使用「影像核心元件」將影像添加到頁面或使用「基礎影像」元件調整大小，則會向某些PNG影像添加具有透明度的灰色邊框(NPR-36879)。
* [!DNL Experience Manager Sites] 管理UI中模板數量多，導航速度慢(NPR-36870)。
* 升級到Service Pack 9會阻止編寫幾個元件。 此問題不允許 [!DNL Sites] 用戶建立新頁面(NPR-36857)。
* 的 `ContextHubImpl` 方法建立 `ResourceResolver` 還沒關。 它會導致有關長期運行的警告消息 `ResourceResolver` 並且該服務有時會返回意外結果(NPR-36853)。
* 在同步藍圖頁面屬性中的單個即時拷貝時，還會同步所有其他即時拷貝(NPR-36829、NPR-36522)。
* 如果僅使用XLS MIME類型，則檔案上載函式不按預期工作(NPR-36785)。
* 帶有Pascal大小寫和所有大寫字的新標籤不會顯示在中的標籤欄位中 [!DNL Content Fragments] (NPR-36742)。
* 添加時的「單文本元素」選項 [!DNL Content Fragment] 導致文本丟失並建立與清單和嵌套清單相關的奇數格式(NPR-36565)。
* 當作者對頁面上的任何元件進行注釋、刪除該元件並對刪除操作執行撤消操作時，嘗試在站點控制台(NPR-36528)中查看該頁面的時間軸資料時，會遇到錯誤。
* 頁面屬性批量編輯器 [!UICONTROL 保存並關閉] 選項保存更改但不關閉編輯器(NPR-36527)。
* 當用戶嘗試將新文本元件拖放到頁面時，該元件會立即消失(NPR-36442)。
* 當用戶在包含空間（系統上不存在的標籤）的按需標籤中鍵入並按Enter時，該標籤將顯示在欄位下。 然而，當 [!DNL Content Fragment] 保存並重新開啟，不顯示按需標籤(NPR-36441)。
* 當通過Dispatcher訪問實例時，無法刪除該模板(NPR-36385)。
* 移動頁面時，需要手動刷新瀏覽器以呈現更改(NPR-36381)。
* 選擇元件時，可以用Ctrl+X或Ctrl+C(和Mac上的Command+X或Command+C)剪切或複製元件。 按一下另一個元件時，可以使用工具欄貼上，但不能使用鍵盤（Ctrl+V或Command+V）(NPR-36379)。
* 當用戶嘗試使用剪刀表徵圖剪切元件以將它們移動到其他位置時，會發生控制台錯誤。 此外，貼上時只移動一個元件(NPR-36378)。
* [!DNL Experience Manager] 在WCM或通知上有沒有索引的查詢，它會降低效能(NPR-36303)。
* 當作者恢復已刪除繼承的元件上的繼承時，可用選項是同步所有頁面內容。 即使僅在一個元件上還原繼承，內容作者仍需要同步完整頁。 完全同步可導致不需要的內容被同步(NPR-34456、CQ-4310183)。
* 「作者」實例上元件的即時使用不顯示所有實例。 某些元件在超過1000頁中使用，但報告只顯示大約40頁(CQ-4323724)。
* 當存在具有大量子頁的站點結構時，在列視圖中載入子頁比在Experience Manager6.5.8中載入Experience Manager(CQ-4322766)花費的時間更長。
* 取消選中「全部」在「展示頁面」選項(NPR-37070)上不起作用。
* 開啟頁面的現有v3元件版本時，不會開啟頁面「屬性」對話框， `NullPointerException` 已記錄(SITES-1830)。

## [!DNL Assets] {#assets-65100}

以下問題已在 [!DNL Assets]:

* 屬性的值 `jcr:title` 在移動資料夾後未在發佈實例上更新。 更名和重新發佈作者中的資料夾不會更新 `jcr:title` 在Publish實例中相同的屬性值(NPR-36369)。

* 如果選擇了兩個或多個資產，並且編輯了一個或多個元資料欄位，則在Safari瀏覽器(NPR-36413)中保存操作將失敗，錯誤代碼為500。

* 由於日期格式不正確，批量元資料導入失敗(NPR-36428)。

* 當在 [!UICONTROL 屬性] 頁面更新元資料時，當架構提供了許多選項時，介面響應速度較慢(NPR-36430)。

* 使用 [!UICONTROL 到期狀態] 謂語無效(NPR-36436)。

* 中各欄位的彈出菜單 [!UICONTROL 資料夾元資料] 屬性不顯示最後選定的值(NPR-36937、CQ-4314429)。

* 搜索檔案和資料夾時，如果用戶應用篩選器並選擇 [!UICONTROL 檔案和資料夾]，只顯示檔案，而不顯示資料夾(CQ-4319543、NPR-36627)。

* 當從資料夾中選擇同一個集合，並且從搜索結果中選擇該集合時，工具欄選項會有所不同(NPR-36620)。

* 的 [!UICONTROL 快速發佈] 選項在搜索結果頁面(NPR-36904、CQ-4317748)中不可用。

* 當用戶在未指定其副檔名的情況下建立資產的即時副本時，下載後該即時副本檔案將不可用(NPR-36903、CQ-4326305)。

* 將用戶添加為子資料夾的所有者時，用戶也會獲得其父資料夾的所有者權限，從而獲得父資料夾的其他子資料夾的所有者權限。 此外，在嘗試刪除父資料夾時，用戶不會作為其所有者被刪除。 (NPR-36801、CQ-4323737)。

* [!DNL Experience Manager] 當您嘗試為複合資產(如PowerPoint演示文稿(NPR-36668))建立子資產時，會生成記憶體不足的異常。

* 當用戶移動已在已發佈站點頁面中使用的資產時，即使未選擇發佈選項，也會再次發佈站點頁面(NPR-36636、CQ-4323500)。

* 使用Apache Tika MIME類型檢測功能時，使用 `AssetManager.createAsset` 方法保留名為 `apache-tika-*.tmp` 檔案。 此臨時檔案使用所有可用的可用磁碟空間(NPR-36545)。

* 下載所有受DRM保護的資產，並且不遵循用戶選擇下載特定資產(CQ-4327422)。

* 無法將資產拖到 `pathfield` (NPR-36849)。

* 在「列」視圖中選擇資產時，資產詳細資訊面板將消失(NPR-36667)。

     <version>6.5.10</version>
