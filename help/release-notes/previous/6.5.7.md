# [!DNL Adobe Experience Manager] 6.5.7.0 {#experience-manager-6570}

[!DNL Adobe Experience Manager] 6.5.7.0是一個重要更新，包括自2019年4月6.5版發佈以來發佈的新功能、客戶要求的關鍵增強功能以及效能、穩定性和安全性改進。 Service Pack已安裝在 [!DNL Adobe Experience Manager] 6.5

中介紹的關鍵功能和增強功能 [!DNL Adobe Experience Manager] 6.5.7.0包括：

* 執行頁面移動和MSM執行非同步操作，以減少它們對運行時效能的影響。

* 用戶可以在「卡」和「列」視圖中對數字資產進行排序。

* [!DNL Assets] 和 [!DNL Dynamic Media] 提供多種輔助功能增強。 這些增強與鍵盤導航、螢幕閱讀器的使用以及使用戶能夠使用類似的輔助技術(AT)有關。 請參閱 [[!DNL Assets] 增強](#assets-6570) 和 [[!DNL Dynamic Media] 增強](#dynamic-media-6570)。

* [表單資料模型HTTP客戶端配置](/help/forms/using/configure-data-sources.md#fdm-http-client-configuration) 以優化效能。

* [每個元件的「重置選項」可用性](/help/forms/using/resize-using-layout-mode.md#resize-components) 在佈局模式下

* [!DNL Experience Manager] 6.5 Service Pack 7Forms提高了以下效能：

   * 在提交自適應表單時驗證伺服器上的欄位值。

   * 使用PDF轉換到自適應表單 [!DNL Automated Forms Conversion service]。

* 支援 [!DNL Microsoft SQL Server] 2019年 [!DNL Experience Manager Forms]。

* 支援 [!DNL Microsoft] SQL Server 2016 Always On可用性組，用於OSGi部署的高可用性。

* 內建存放庫 (Apache Jackrabbit Oak) 更新至 1.22.5 版。

以下是中提供的修復清單 [!DNL Experience Manager] 6.5.7.0版。

## [!DNL Sites] {#sites-6570}

* 開啟 [!UICONTROL 時間繞排] 選項，保持時間軸側滑軌選項處於開啟狀態，然後導航至 [!UICONTROL 站點] 控制台 `Failed to Load` 出現錯誤(NPR-34951)。

* 當頁面的路徑包含多個內容子字串時，預覽無法呈現，版本比較函式也失敗(NPR-34740)。

* 為 `String` 鍵入元件的label屬性，可以刪除元件並撤消刪除操作。 但是，撤消刪除後，標籤屬性將從 `String` 至 `Long` (NPR-34739)。

* 在將基於具有鎖定佈局的模板的體驗片段添加到頁面(NPR-34632)時發生以下異常：

   ```TXT
   org.apache.sling.api.SlingException: Cannot get DefaultSlingScript: org.apache.sling.api.SlingException: Cannot get DefaultSlingScript: org.mozilla.javascript.EcmaError: TypeError: Cannot call method "getChildren" of null
   ```

* 移動資料夾時，會導致遍歷問題，並出現以下錯誤(NPR-34554):

   ```TXT
   org.apache.sling.api.SlingException: Cannot get DefaultSlingScript. org.apache.jackrabbit.oak.query.RuntimeNodeTraversalException: The query read or traversed more than 100000 nodes. To avoid affecting other tasks, processing was stopped
   ```

* 建立、發佈新資產並將其移動到新位置時， `Request to complete move operation` 工作流將被建立，並且結果處於「中止」狀態。 上載新資產並執行 `move` 操作導致建立 `Request to complete move operation` 工作流處於掛起狀態(NPR-34543)。

* 當您在彈出式菜單中選擇項目並更新所需檔案時，它不允許保存對話框值，因為另一個菜單的必填欄位為空(NPR-34529)。

## [!DNL Assets] {#assets-6570}

[!DNL Adobe Experience Manager] 6.5.7.0 [!DNL Assets] 修復了以下問題並提供了以下增強功能。

* 對中的輔助功能執行了以下增強 [!DNL Experience Manager Assets] 在本版。 有關詳細資訊，請參見 [輔助功能 [!DNL Assets]](/help/assets/accessibility.md)。

   * 使用鍵盤導航時間軸時， `Esc` 鍵可以折疊 [!UICONTROL 全部顯示] (CQ-4293598)。
   * 使用鍵盤標籤鍵導航時，在從添加的標籤中刪除最後一個標籤後，標籤欄位將保留焦點(NPR-35109)。
   * [!DNL Experience Manager] 元件現在包含供螢幕閱讀器使用的名稱、角色和值的適當資訊(NPR-34255)。
   * 刪除「類型/大小」組合框、「連結」組合框、「語言」組合框或「文本編輯」框後，鍵盤焦點將返回下一個或上一個用戶介面元素或更相關的用戶介面元素(CQ-4293585)。
   * 將指針懸停在選項上時，將顯示「選擇」和「下載」等提示。 使用螢幕放大鏡的用戶可能看不到檔案縮略圖，因為這些提示。 現在，在使用 `Escape` 按鈕 (CQ-4293554).
   * 在從頁面中存在的網格中選擇網格單元時，焦點將移到螢幕上顯示的操作欄(CQ-4282127)。
   * 可視用戶可以區分普通文本和連結，因為顯示視覺線索（下划線和雪佛龍表徵圖），以便連結到中的所有解決方案 [!DNL Experience Manager] 首頁(CQ-4282072)。

* 以下用戶體驗增強在 [!DNL Assets]:

   * 啟用對卡視圖和列視圖中的資產進行排序(NPR-35097)。

* 升級到6.5後，如果使用Assets HTTP API生成JSON檔案，則檔案(NPR-35129)中使用的編碼存在問題。

* 未提供建立集合權限（「建立集合」選項不可用）的組的用戶仍然能夠通過直接訪問URL來建立集合 `https://[aem_server]:[port]/mnt/overlay/dam/gui/content/collections/createcollectionwizard.html/content/dam/collections?contentPath=/content/dam/collections` (NPR-35115)。

* 按名稱排序時，搜索的資產按區分大小寫的方式排序。 這基於在搜索結果中以有序方式出現的大小寫建立兩個單獨的排序清單(NPR-35068)。

* 在編輯器中開啟內容片段時，將顯示警告消息(`Invalid value specified for a metadata property`)記錄在錯誤日誌中(NPR-35012)。

* 沒有管理員權限的用戶可以使用 [Experience Manager] 案頭應用。 (NPR-34993).

* 當在Assets用戶介面上拖動同一Asset並建立新版本時，元資料中的更改不是持久的(NPR-34940)。

* 編輯集合時，用戶可以刪除集合的標題並成功保存更改(NPR-34889)。

* 上載重複映像時，將顯示刪除選項。 選擇「刪除」可上載影像。 還觸發了DAM更新資產工作流(NPR-34744)。

* 使用時 [!DNL Adobe Asset Link] 與 [!DNL Adobe InDesign]，搜索結果不包含資料夾和集合，但僅包含資產(NPR-34699、CQ-4303666)。

* 將指針懸停在卡視圖上，使螢幕滾動（自動）聚焦於卡中可用的快速操作(NPR-34514)。

* 批量編輯多個資產的屬性時，選擇 [!UICONTROL 保存] 選項關閉批量編輯器視圖並重定向到主 [!DNL Assets] 的子菜單。 此行為與 [!UICONTROL 保存並關閉] 選項，而不應為(NPR-34546)。

* 保存後，智慧集合不顯示正確的用戶介面設定。 查詢已正確保存，但介面始終顯示上次添加的Option謂詞(NPR-34539)。

* 將資產添加到 [!DNL Experience Manager]，沒有命名空間的元資料不會被導入(NPR-34530)。

* 在資料夾上拖動資產以移動它時，用戶介面還顯示 [!UICONTROL 放入光箱] 和 [!UICONTROL 放入集合]。 即使取消移動操作，用戶介面仍繼續顯示後兩個選項(NPR-34526)。

* 符號 `%>` 顯示在收藏頁(NPR-34499)上。

* 在列視圖中， [!DNL Assets] 在向上和向下滾動時顯示重複的資料夾和資產名稱，然後顯示所有資產(NPR-34464)。

* 如果在建立公用資料夾後立即建立專用資料夾，則公用資料夾將使用專用資料夾設定(NPR-34415)。

* 在卡視圖中，卡未按字母順序列出，卡無法按字母順序排序(NPR-34234)。

* 重新開啟級聯規則時，不會在用戶介面(CQ-4301452)上維護選項。

## UberJar {#uber-jar}

UberJarExperience Manager6.5.7.0在 [Maven中央儲存庫](https://repo.maven.apache.org/maven2/com/adobe/aem/uber-jar/6.5.7/)。

要在Maven項目中使用UberJar，請參見 [如何使用UberJar](/help/sites-developing/ht-projects-maven.md) 並在項目POM中包括以下依賴關係：

```shell
<dependency>
     <groupId>com.adobe.aem</groupId>
     <artifactId>uber-jar</artifactId>
     <version>6.5.7</version>
     <scope>provided</scope>
</dependency>
```

>[!NOTE]
>
>UberJar和其他相關工件可在Maven Central Repository（Maven中央儲存庫）上找到，而不是AdobePublic Maven儲存庫(`repo.maven.apache.org`)。 主UberJar檔案已更名為 `uber-jar-<version>.jar`。 所以，沒有 `classifier`, `apis` 值，對於 `dependency` 標籤。
