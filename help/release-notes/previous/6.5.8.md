# [!DNL Adobe Experience Manager] 6.5.8.0 {#experience-manager-6580}

[!DNL Adobe Experience Manager] 6.5.8.0包括自2019年4月6.5版發佈以來發佈的新功能、客戶要求的關鍵增強功能以及效能、穩定性和安全性改進。 Service Pack已安裝在 [!DNL Adobe Experience Manager] 6.5

中介紹的關鍵功能和增強功能 [!DNL Adobe Experience Manager] 6.5.8.0是：

<!-- TBD:
* Using the Connected Assets functionality, it is now possible to connect up to 3 [!DNL Sites] instances with 1 [!DNL Assets] instances. The configuration user interface now allows the administrators to provide the details of these [!DNL Sites] instances. -->

* 使用時 [連接的資產功能](/help/assets/use-assets-across-connected-assets-instances.md)，現在可以查看所有 [!DNL Sites] 頁。 資產的這些引用可在 [!UICONTROL 屬性] 的子菜單。 這使管理員、營銷人員和圖書管理員能夠完整地查看資產使用情況，從而實現更好的跟蹤、管理和品牌一致性。

* 刪除網頁中引用的資產時， [!DNL Experience Manager] [顯示警告](/help/assets/use-assets-across-connected-assets-instances.md#asset-usage-references)。 可以強制刪除參照的資產，或檢查和修改在 [!DNL Properties] 的子菜單。 按一下參照可開啟本地和遠程 [!DNL Sites] 頁。

* 使用 [!UICONTROL 名稱]。 [!UICONTROL 上次修改日期，] 和 [!UICONTROL 上次推廣日期] 屬性。

* 內置儲存庫(Apache Jackrabbit Oak)已更新為1.22.6。 <!-- TBD: Mention the version -->

以下是中提供的修復清單 [!DNL Experience Manager] 6.5.8.0版。

## [!DNL Sites] {#sites-6580}

* 當將頁面移到藍圖時，不更新鏈路的目的地(NPR-35724)。
* 基於Tizen的播放器無法在某些瀏覽器上進行身份驗證。 不支援samesite=none屬性(NPR-35589)的瀏覽器出現問題。
* 解鎖響應容器不顯示允許的元件(NPR-35565)。
* 當您建立新添加的頁面的即時副本時，語言母版將為每個域建立兩個副本(NPR-35545)。
* SCR元件註冊表中由於 `org.apache.felix.scr.impl.ComponentRegistry` 計時器。 因此， [!DNL Experience Manager] 無限期停止響應(GRANITE-33125,FELIX-6252)。
* 當您在側軌中搜索特定資產時，結果包含一些未搜索的資產(NPR-35524)。
* 為Experience Manager實例啟用SSL時，上下文路徑將被刪除(NPR-35477)。
* 建立清單時，添加一些文本作為第一個元素，添加一個表作為第二個元素，並在表內添加一個清單，父清單會扭曲(NPR-35465)。
* 在連續清單項上使用不同插件時， <br> 標籤被添加到清單項(NPR-35464)。
* 當清單置於兩段之間時，不能將表添加到清單(NPR-35356)。
* 從6.3AEM到AEM6.AEM5的實例升級時，升級實例的啟動時間較長(NPR-35323)。
* 複製包AEM含括弧()的資產時。 在名稱中，複製失敗(GRANITE-27004、NPR-35315)。
* 將標題添加到富格文本編輯器時，段落按鈕將被禁用(NPR-35256)。
* 將項目添加到現有清單時，它會刪除後續的可折疊清單或切換清單(NPR-35206)。
* 選擇「展示」頁選項後，將顯示包含所有可用即時副本的對話框，並進行自動展示。 頁面的即時拷貝將在不使用用戶操作的情況下推廣到所有地理區域(NPR-35138)。
* 使用「包括子項」選項時，「管理發布」選項不列出所有頁。 僅列出22頁(NPR-35086)。
* 編輯策略時，文本元件不會保留策略更改(NPR-35070)。
* 在編號清單中縮進某些項目時，所有項目都保持相同的編號，但對於具有相同縮進的項目，編號應從1開始(CQ-4313011)。
* 啟用微型化後，您將無法編輯任何頁面或元件。 問題在安裝AEM6.5 Service Pack 7(CQ-431133)後開始。
* Omni搜索和資產過濾器返回無關或無結果(CQ-4312322,NPR-35793)。
* 當多個頁面同時訪問客戶端庫時，HTML庫管理器無法載入客戶端庫。 這導致頁面呈現不正確(NPR-35538)。
* 在中設定SSL時，上下文路徑將自動刪除 [!DNL Experience Manager] (NPR-35294)。
* 在按一下「註銷」選項(NPR-35160)後，包管理器不會註銷用戶。

## [!DNL Assets] {#assets-6580}

* Handlebars JavaScript庫已升級到v4.7.6(NPR-35333)。
* 當您從批量元資料編輯開始並取消選擇項目，直到仍然選擇單個項目(NPR-35144)時，元資料編輯器介面將停止按預期工作。

## UberJar {#uber-jar}

UberJarExperience Manager6.5.8.0在 [Maven中央儲存庫](https://repo.maven.apache.org/maven2/com/adobe/aem/uber-jar/6.5.8/)。

要在Maven項目中使用UberJar，請參見 [如何使用UberJar](/help/sites-developing/ht-projects-maven.md) 並在項目POM中包括以下依賴關係：

```shell
<dependency>
     <groupId>com.adobe.aem</groupId>
     <artifactId>uber-jar</artifactId>
     <version>6.5.8</version>
     <scope>provided</scope>
</dependency>
```

>[!NOTE]
>
>UberJar和其他相關工件可在Maven Central Repository（Maven中央儲存庫）上找到，而不是AdobePublic Maven儲存庫(`repo.maven.apache.org`)。 主UberJar檔案已更名為 `uber-jar-<version>.jar`。 所以，沒有 `classifier`, `apis` 值，對於 `dependency` 標籤。

## Experience Manager6.5.8.0中的已知問題 {#known-issues-6580}

* 如果要升級 [!DNL Experience Manager] 實例從6.5版到6.5.8.0版，您可以查看 `RRD4JReporter` 例外情況 `error.log` 的子菜單。 重新啟動實例以解決問題。

* 如果安裝 [!DNL Experience Manager] 6.5 Service Pack 5或上一個Service Pack [!DNL Experience Manager] 6.5，資產自定義工作流模型的運行時副本(建立於 `/var/workflow/models/dam`)。
要檢索運行時副本，Adobe建議使用HTTP API將自定義工作流模型的設計時副本與其運行時副本同步：
   `<designModelPath>/jcr:content.generate.json`。

* 聯繫Adobe客戶支援（如果在中編輯和建立級聯規則時遇到問題） [!UICONTROL 資料夾元資料架構Forms編輯器] 和 [!UICONTROL 元資料架構Forms編輯器] 使用 [!UICONTROL 定義規則] 對話框。 已建立和保存的規則正按預期工作。

* 當用戶首次以自適應格式選擇配置欄位時，「屬性瀏覽器」中不會顯示保存配置的選項。 選擇以在同一編輯器中配置自適應表單的其它一些欄位可解決此問題。
