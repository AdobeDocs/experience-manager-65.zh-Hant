# [!DNL Adobe Experience Manager] 6.5.9.0 {#experience-manager-6590}

[!DNL Adobe Experience Manager] 6.5.9.0包括自2019年4月6.5版發佈以來發佈的新功能、客戶要求的關鍵增強功能以及效能、穩定性和安全性改進。 Service Pack已安裝在 [!DNL Adobe Experience Manager] 6.5

中介紹的關鍵功能和增強功能 [!DNL Adobe Experience Manager] 6.5.9.0是：

* [!DNL Experience Manager Sites] Dynamic Media基金會元件現在允許您在使用響應影像預設或智慧裁剪時開啟或關閉高解析度設備的優化。

* 為了提高效能， `hidden=false` 條件從JCR查詢移到 [!UICONTROL 查詢生成器] 計算器。 要驗證在更改後隱藏的謂詞是否正在工作， [!DNL Experience Manager] 檢查是否未顯示任何隱藏資料夾。

* 能夠恢復刪除的頁面和樹 [!DNL Experience Manager Sites] 的子菜單。

* 支援新用戶使用郵件程式配置服務的刷新令牌刷新訪問令牌。

* [支援SMTP XOAUTH2](/help/sites-administering/notification.md#setting-up-oauth) 郵件配置服務的機制。

* 支援 [!DNL MongoDB] 版本4.2和4.4。

* 與香港、澳門和台灣有關的名稱的出現將根據中國地區和地區的新命名慣例進行更新。

* 在 [!DNL Experience Manager] [[!DNL Assets]](#assets-accessibility-6590) 和 [[!DNL Dynamic Media]](#accessibility-dm-6590)。

* 智慧成像DPR（設備像素率）和網路頻寬優化使您能夠高效地提供最優質的影像；在解析度高且網路頻寬受限的設備上。 有關詳細資訊和時間表，請參見 [智慧映像常見問題](/help/assets/imaging-faq.md)。

* [!DNL Dynamic Media] 交貨(D)`fmt` URL修飾符)支援下一代影像格式AVIF（AV1影像格式）。 有關詳細資訊和時間表，請參見 [影像服務和呈現API fmt](https://experienceleague.adobe.com/docs/dynamic-media-developer-resources/image-serving-api/image-serving-api/http-protocol-reference/command-reference/r-is-http-fmt.html)。

* 能夠使用 [!UICONTROL 分配任務] 工作流步驟。

* 修改源交互通信後檢索交互通信草稿的能力。

* 設定自定義域名，以在中載入、呈現和驗證reCAPTCHA服務 [!DNL Experience Manager Forms]。

* 在中選擇表單資料模型和服務時 [!UICONTROL 調用表單資料模型服務] 工作流步驟，為輸入資料指定服務參數。

如果選擇 [!UICONTROL 相對於負載] 選項以附加檔案作為服務參數，您現在可以指定包含該檔案的資料夾路徑而不是實際檔案名。 定義資料夾名稱（而不是檔案附件名稱）使您能夠重用工作流模型。 不將工作流模型限制為單個檔案附件名稱。

* 能夠在中的「記錄文檔」模板中使用多個母版頁 [!DNL Experience Manager Forms]。

* 支援中記錄文檔中的分頁符 [!DNL Experience Manager Forms]。

* 內置儲存庫(Apache Jackrabbit Oak)已更新為1.22.7。

## UberJar {#uber-jar}

UberJarExperience Manager6.5.9.0在 [Maven中央儲存庫](https://repo.maven.apache.org/maven2/com/adobe/aem/uber-jar/6.5.9/)。

要在Maven項目中使用UberJar，請參見 [如何使用UberJar](/help/sites-developing/ht-projects-maven.md) 並在項目POM中包括以下依賴關係：

```shell
<dependency>
     <groupId>com.adobe.aem</groupId>
     <artifactId>uber-jar</artifactId>
     <version>6.5.9</version>
     <scope>provided</scope>
</dependency>
```

>[!NOTE]
UberJar和其他相關工件可在Maven Central Repository（Maven中央儲存庫）上找到，而不是AdobePublic Maven儲存庫(`repo.maven.apache.org`)。 主UberJar檔案已更名為 `uber-jar-<version>.jar`。 所以，沒有 `classifier`, `apis` 值，對於 `dependency` 標籤。
